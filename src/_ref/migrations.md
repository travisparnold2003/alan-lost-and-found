# Alan Migration Reference

> **CRITICAL RULE — HARDEST WON LESSON**
> ALL collections in migration.alan require `<! !>` before `( )`.
> - WRONG: `'Airlines': collection = ( )`
> - RIGHT: `'Airlines': collection = <! !> ( )`
> - This applies to BOTH from_empty AND from_release migrations.
> - Empty `( )` alone causes: `error: expected keyword '<!'' but saw keyword '('`
> - This is different from everything else in Alan — it is specific to migration collection syntax.

Migrations transform an existing dataset when the model changes. Every deploy after the first requires a valid `migration.alan`.

---

## When Migrations Run

- **First deploy:** uses `migrations/from_empty/` (maps from empty to initial model)
- **All subsequent deploys:** uses `migrations/from_release/` (maps from last deployed model to new model)

The `from/` folder in `from_release/` is auto-generated by `Alan: Generate Migration` and contains a snapshot of the previously deployed model.

---

## Generate Migration

From the Command Palette (`Ctrl+Shift+P` / `Cmd+Shift+P`):
```
Alan: Generate Migration
```

Select **from_release** to generate a migration from the current live version.

This auto-fills mappings for unchanged properties. You only need to handle:
- Renamed properties
- Restructured data (e.g., splitting a field, merging two)
- New required properties (add defaults)
- Removed properties (leave out or keep with `<!""!>` to mark as intentionally dropped)

---

## Migration Syntax

### Root structure
```alan
root = root as $ (
	// ... map every base property
)
```

The `$` variable holds the current node from the old model.

### Text mapping
```alan
'Name': text = $ .'Old name'             // rename
'Description': text = ""                 // new property, empty default
'Label': text = "Default label"          // new property, static default
'Code': text = $ .'Code'                 // unchanged (identity mapping)
```

### Number mapping
```alan
'Price': number = $ .'Price'             // unchanged
'Year': number = 2024                    // new with numeric default
'Score': number = 0
```

### File mapping
```alan
'Document': file = $ .'Document'
```

### Group mapping
```alan
'Address': group = $ .'Address' (
	'Street': text = $ .'Street'
	'City': text = $ .'City'
)
```

### Collection mapping (rename or restructure)
```alan
'Products': collection = map $ .'Old products'* as $'prod' (
	'SKU': text = $'prod' .'Product code'       // old key → new key
	'Name': text = $'prod' .'Product name'
	'Price': number = $'prod' .'Price'
	'Stock': number = 0                          // new property
)
```

### Stategroup mapping
```alan
// Unchanged stategroup
'Status': stategroup = $ .'Status' (
	'Active': $ .'Status'?'Active' ( )
	'Inactive': $ .'Status'?'Inactive' ( )
)

// New states added (old states still map)
'Status': stategroup = $ .'Status' (
	'Active': $ .'Status'?'Active' ( )
	'Pending': $ .'Status'?'Pending' ( )        // new state
	'Closed': $ .'Status'?'Active' ( )          // merge old Active into Closed
	'Archived': 'Archived' (                     // default new entries to Archived? No — use none
	)
)

// Stategroup with state-specific properties
'Type': stategroup = $ .'Type' (
	'Simple': $ .'Type'?'Simple' ( )
	'Complex': $ .'Type'?'Complex' as $'c' (
		'Complexity level': text = $'c' .'Level'
	)
)
```

### Nested collections
```alan
'Orders': collection = map $ .'Orders'* as $'order' (
	'Order ID': text = $'order' .'Order ID'
	'Lines': collection = map $'order' .'Lines'* as $'line' (
		'Line': text = $'line' .'Line'
		'Quantity': number = $'line' .'Quantity'
		'Product': text = $'line' .'Item'        // renamed reference
	)
)
```

## Optional Error Messages

Mark intentionally incomplete mappings to suppress warnings:
```alan
'Legacy field': text = <!"This field is being dropped in this release"!>
```

## Common Migration Scenarios

### Rename a collection key
```alan
// Old: 'Product code', New: 'SKU'
'Products': collection = map $ .'Products'* as $'p' (
	'SKU': text = $'p' .'Product code'     // new key name
	// ... rest of properties
)
```

### Split one field into two
```alan
// Old: 'Full name', New: 'First name' + 'Last name'
'First name': text = ""      // can't split automatically — set empty, fix data after deploy
'Last name': text = $ .'Full name'   // put full name in last name as fallback
```

### Merge two fields into one
```alan
'Full name': text = concat ( $ .'First name' , " " , $ .'Last name' )
```

Wait — `concat` is an expression available in migrations:
```alan
'Display name': text = concat ( $ .'First' , " " , $ .'Last' )
```

### Change from flat to nested (group)
```alan
'Contact': group = (                     // new group wrapper
	'Email': text = $ .'Email'           // was at root level
	'Phone': text = $ .'Phone'
)
```

### Move property between states
```alan
'Type': stategroup = $ .'Status' (
	'Active': $ .'Status'?'Open' as $'s' (
		'Due date': number = $'s' .'Deadline'   // property moved into this state
	)
	'Closed': $ .'Status'?'Closed' ( )
)
```

### Add new required collection entry (bootstrap data)
Use `password-initializer` or the migration initializer to seed required entries. For general data, add it manually after deploy via the UI or use a `from_empty` migration that sets initial values.

---

## Migration Constraints

- **All base properties must be mapped** — you cannot omit them even if you plan to drop them
- **Derived properties are NOT migrated** — they are recomputed automatically
- **Cannot perform complex transforms natively** — workaround: add derived properties to the old model snapshot before migration, then read from those derived values
- **Collection order is not guaranteed** — do not rely on insertion order

---

## from_empty Migration Pattern

> The from_empty migration runs on first deploy to seed initial data.
> MUST use `<! !>` for all collections (see critical rule at top of this file).

For first deploy with empty collections (user fills data via UI):
```alan
root = root as $ (
	'Categories': collection = <! !> ( )
	'Products': collection = <! !> ( )
)
```

With pre-seeded data:
```alan
root = root as $ (
	'Airlines': collection = <! !> (
		= ( 'IATA Code': text = "KL"  'Name': text = "KLM Royal Dutch Airlines" )
		= ( 'IATA Code': text = "BA"  'Name': text = "British Airways" )
	)
	'Terminals': collection = <! !> (
		= ( 'Code': text = "MAIN"  'Name': text = "Main Terminal Building" )
	)
	'Locations': collection = <! !> (
		= ( 'Name': text = "Departure Hall 1"  'Terminal': text = "MAIN"  'Area': stategroup = 'Departure Hall' ( ) )
	)
	'Item Categories': collection = <! !> (
		= (
			'Category': text = "Electronics"
			'Subcategories': collection = <! !> (
				= ( 'Subcategory': text = "Smartphone" )
				= ( 'Subcategory': text = "Laptop" )
			)
		)
	)
)
```

Key rules:
- `= ( ... )` is one entry
- Stategroup values in migrations: `stategroup = 'StateName' ( )`
- Nested collections also need `<! !>`
- The key field (e.g. `'IATA Code'`) must be the first field listed
